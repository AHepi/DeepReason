# Jolt architecture inquiry

DeepReason currently asks models to perform some work that may belong to the
harness. The inquiry must determine the actual failure from repository evidence,
not presume that every prompt instruction should become code or that all model
discretion is harmful.

The architectural question is how to separate deterministic orchestration,
bounded model reasoning, evidence acquisition, formal epistemic state,
non-authoritative scratch work, criticism, typed repair, phase transitions,
stopping, and grounded final composition while preserving open-ended search.

The inquiry distinguishes implemented behaviour, documented intent, historical
proposal, test-backed invariant, unsupported assumption, and future architecture.
It evaluates rival architectures against C1 through C15 from the run directive,
requires falsifiable tests, treats LLM criticism as scrutiny rather than truth,
and permits partial or unresolved conclusions.

The requested four-route topology is frozen as follows: GLM 5.2 for the systems
school, DeepSeek v4 Pro for adversarial analysis, Qwen 3.5 397B for typed-contract
analysis, and Kimi K2.6 for integration and migration analysis. The current
manifest schema binds routes to roles rather than schools; the experiment driver
must therefore record every auxiliary role-to-school use explicitly instead of
pretending that native school-route binding exists.
