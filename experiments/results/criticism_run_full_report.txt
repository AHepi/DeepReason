=====================================================================
1. WHAT DEEPREASON IS (primer)
=====================================================================
DeepReason is a deterministic "conjecture-criticism" epistemology harness
(~6.5k lines of Python). An LLM is used ONLY as a bounded pure function
(prompt pack -> schema-validated JSON); all epistemology lives in the
deterministic harness. Core invariants: an append-only event log is the
source of truth (replay reproduces state byte-for-byte); nothing is ever
deleted; artifact status (accepted/refuted) is computed by a Dung-style
grounded-extension adjudicator over attack/dependence edges carried by
WARRANTS — measures and scores steer attention but never decide status.

A run cycles: a CONJECTURER emits candidate artifacts (JSON "skeletons"
with claim/mechanism/scope/forbidden-cases); an anti-relapse gate blocks
candidates battery-equivalent to already-refuted ones; CRITICISM attacks
survivors three ways (deterministic program checks; an argumentative
critic; and for informal standards, a full TRIAL: critic drafts the case,
a defender answers, a two-model judge ensemble rules, and the ruling is
screened by referential-integrity, order-swap, and paraphrase checks —
only screened rulings become warrants); ADJUDICATION then recomputes all
statuses. Refuted candidates spawn successor problems; rival survivors
spawn discrimination problems; "schools" (persistent stances, e.g.
mechanist vs skeptic) condition generation. Judges are themselves audited
(paraphrase-invariance and premise-deletion audits can refute a ruling's
validity node, reinstating its target).

=====================================================================
2. THE OPEN QUESTION AND PRIOR MEASURED DATA
=====================================================================
Question: does harness-style criticism (an adversarial critic filters a
candidate pool before majority vote) extract more reliable answers than
self-consistency (unfiltered majority vote over the SAME pool)?

Three validation studies (validate.py; k=5 candidates/question, one
shared pool per question, answer key held out from every model call):

  easy set (24 q), deepseek-v4-pro:  single=SC=harness=1.00 (ceiling)
  easy set (24 q), deepseek-v4-flash: all arms 0.958; critic precision
      0.333, recall 0.25 at candidate base error 0.033
  hard set (20 q), deepseek-v4-pro:  single 0.95, SC 1.00, harness 1.00;
      base error 0.01; critic recall 1.0, precision 0.125;
      net fixed-minus-broke = 0

Verdict so far: UNDECIDABLE — every arm ceilinged. The critic showed
perfect recall but 7 false flags per true flag; 4/5 majorities absorbed
the noise. The question needs a regime where majorities are contested.

=====================================================================
3. THIS EXPERIMENT: THE HARNESS WORKS ITS OWN QUESTION
=====================================================================
Following the repo's self-referential pattern (it previously designed its
own cache layer and self-calibration), the question was posed TO the
harness as a seeded problem:

  Standard std-crit-design (rubric, absolute mode): proposals must
  (1) name the regime condition in measurable quantities (base error
  rate, critic precision/recall, k); (2) state forbidden cases that are
  measurable outcomes of running the proposed experiment; (3) respect
  study-design honesty (held-out key, pre-registered thresholds, one
  shared pool per question).

  Problem pi-criticism: carries the measured data above verbatim and
  asks for the regime prediction — including how filtering can LOSE when
  false flags erode correct majorities — plus a decisive experiment
  executable within a 300k-token budget.

Config: deepseek-v4-pro (all generation roles) + deepseek-v4-flash as the
second judge seat (cross-model ensemble); 2 schools (mechanist, skeptic);
40 cycles; hard token budget 700,000.

If the critic is a superforecaster but its errors are independent of the model's confusion zones, it may not help.
