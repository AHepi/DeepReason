# Experiment record index, 2026-07-13

This index supersedes `INDEX_2026-07-05.md`. It records one act: the
retirement of the pre-rebuild experiment record from the working tree.

## Retirement of the pre-rebuild result files

**What happened.** The 61 pre-rebuild result files under
`experiments/results/` (reports, survivor lists, rankings, comparisons) were
removed from the working tree in the commit that adds this index. Retained,
because they are current evidence for the rebuilt harness, not history:

- `INDEX_2026-07-05.md` (indexes are append-only)
- `gemma4_dna_unattended_report.json` and
  `gemma4_dna_unattended_3_report.json` (live small-model run evidence;
  inputs to experiment E1.2)
- `small_model_compat_local_verification_v1.json` (the compat kernel's
  local verification record)

The gemma run roots (`experiments/gemma4_dna_unattended_2026-07-12/` and
`experiments/gemma4_dna_unattended_3_2026-07-12/`) are also retained: they
are the in-repo replayable roots that satisfy the byte-equal replay check in
`docs/SELF_IMPROVEMENT.md` step 5. The prereg YAMLs, compat matrix and
baseline JSONs, validation question sets, and design notes under
`experiments/` are retained as the methodology's contracts and reusable
inputs.

**Why.** Operator decision: the harness has been substantially rebuilt on
this branch, and the pre-rebuild records, while still compatible with the
new build, no longer guide improvement. The repository is the harness; when
DeepReason runs as a plugin, run logs are saved at the operator's site, not
here. Experiment records are kept in-repo only while they guide improvement.

**This is a supersession, not a deletion.** Per `docs/SELF_IMPROVEMENT.md`
(nothing is deleted; supersede, don't replace), the full record remains
byte-exact in git history. Recover it at any time with:

```bash
git checkout 3d839b3 -- experiments/results/
```

Commit `3d839b3` ("tests: verify all workload roots and memory attachments")
is the last commit containing the complete record and remains reachable as
the base of this branch.

**Citations elsewhere.** `INDEX_2026-07-05.md` and the narrative docs
(`README.md`, `docs/CAN_LLMS_EXPLORE.md`, `docs/BASIN_REPORT.md`,
`docs/STATE_OF_THE_THEORY.md`, `docs/MINI_PLAN.md`, and others) cite result
files by path. Those citations refer to the archived record at the commit
above; the claims they back are unchanged.

**Effect on the active experiment program.** Tier 0 of
`docs/EXPERIMENT_PROGRAM_2026-07.md` (E0.1, E0.3) uses historical data as
zero-token calibration corpora. That data is recovered from the archive
commit when those experiments run; the gemma roots retained here are the
in-repo replayable corpus and are unaffected by this retirement.

## Standing guidance

New experiments append their result files here and add a line to the latest
index, exactly as before. An experiment that isn't in an index does not
exist. Retirement of future superseded records should follow the pattern of
this entry: remove from the working tree, record the recovery SHA, never
rewrite history.

## Experiments

- **E0.1 real-embedder recalibration** (zero-token): all four preregistered
  predictions REFUTED. Hash-vs-neural novelty rank correlation 0.51/0.49
  (line: 0.5); hash-novel contamination by neural near-duplicates 1.0/1.0
  (falsifier: 0.40); neural embedder not duplicate-vs-sibling separable on
  website corpora; calibrated reseed thresholds never fire on the
  recoverable corpus. Headline: under a real embedder the entire gemma
  conjecture stream is more homogeneous than typical paraphrase pairs, so
  hash-scale "variation" was largely near-duplication. Consequences: prior
  hash-based novelty numbers demoted to unverified; E2.3 now gates any
  repetition of the soft-basin claim. Caveats: n=2 roots, both gemma4:31b
  website runs. Prereg: `experiments/e01_embedder_recalibration_prereg.yaml`.
  Report: `experiments/results/e01_embedder_recalibration_report.json`.
